=head2 Range descritption

                  format                complete range example
		  
  normal range => integer               header, 4, bright_blue     
  comment      => #                     data section start, # 
  bitfield     => [xInteger]bInteger    bitfield, x8b4 
  skip range   => xInteger              boring, x256,, a comment

=head3 Direct command line defintion

You can define your range description on the command line by passing a string to  the I<--range_description> (short I<-r>) option.

  hdr -r 'magic, 5,   :other,37  :bf,b8   :skip,x32,, I skipped :more, 20'

=head3 Definition from a file

The I<--range_description> option also accept a file namewhich contain a range description written in Perl.

  hdr -r my_range_description_file

=head4 simple file

 	[
 	  ['data header', 5, 'blue on_cyan'],
 	  ['data', 20, 'blue on_bright_yellow'],
 	] ;

=head4 more structured file

 use strict ;
 use warnings ;
 
 # any perl code you may need to generate your range definition
 
 my $data_range = # definition to re-use
 	[
 	  ['data header', 5, 'blue on_cyan'],
 	  ['data', 20, 'blue on_bright_yellow'],
 	] ;
 
 my $structured_range = 
 	[
 	  ['zero size', 0],
 	  [
 	    ['magic cookie', 12, 'red'],
 	    ['padding', 32, 'yellow'],
 	    $data_range, 
 	  ],
          ['A comment', '#'],		
 	  [
 	    ['extra data', 18, undef],
 	      [
 	      $data_range, 
 	      ['footer', 4, 'bright_yellow on_red'],
  	    ]
   	  ],
  	] ; 
 	
 $structured_range ; # return the range description

See Also L<Advanced usage>

=head3 Defining your own colors

Define a custom color file containing a data in the format below andd pass it as argument to option I<--colors>

  {
  HTML =>
	{
	# custom color name
	custom => 'color:#f00; background-color:#ff0; ',
	
	# the normal ansi color
	reset => '',
	white => 'color:#aaa; ',
	black => 'color:#000; ',
	green => 'color:#0a0; ',
	...
	},
  ANSI =>
	{
	# custom color name, it's a good idea to define them on both formats
	custom => 'bright_red on_bright_yellow; ',
	
	reset => '',
	white => 'white',
	...
	},
  }

=head2 Input 

=head3 from file or stdin

  hdr -r 'magic cooookie, 5,  ...'  -rul -o ver -col my_file

or pipe your data into I<hdr>

  cat my_file | hdr -r 'magic, 5,  ...' -rul -o ver -col

=head3 Too little data

I<hdr> will display one or more warnings is there is not enough data for all the defined ranges.

  Warning: range 'xxx' size 32 was shortened to 8 due to maximum size limit
  Warning: More ranges to display but no more data.

=head3 Too much data

I<hdr> only uses the  part that is used by your range descriptions. The rest is ignored.

If you want to use an existing range description that contains many range definitions, you can use the I<--maximum_size> option
to limit the amount of data I<hdr> will use.

  $>hdr -r 'A,50:B,50:C,50:D,50' --maximum_size 100

will display a warning:

  Warning: More ranges to display but no more data.

=head3 dynamic range generation

The parser below will alternatively return a 'from 0' or 'from 1' range description depending on the first byte of the data to display

 sub my_parser 
	{
	my ($dumper, $data, $offset) = @_ ;
	
	my $first_byte = unpack ("x$offset C", $data) ;
	
	$offset < length($data)
		?  $first_byte == ord(0)
			? ['from "0"', 5, 'bright_yellow']
			: ['from "1"', 3, 'bright_green']
		: undef ;
	}	
 my $data = '01' x 20 ;
 print $hdr->dump(\&my_parser, $data) ;

=head3 Comments that help

Remember that you can, while generating your ranges, assign the comment field

  [range_name, 4, red, this comment can be of great help (but will truncated)]
  
You can also pass a sub reference in place of the comment. The sub will be called just before the range is displayed. It is passed information
about the data to be displayed (data and offset). This allows you to dynamically generate a comment string.

=head2 Let the computer do the heavy lifting

B<hdr> may be a nice tool to run from the command line but you can also use it in shell scripts to generate reports. I generally write I<pod> so I let I<hdr> generate my pod 
snippets with embeded html hexdumps.

 printf "=head1 MY REPORT\n\n" > report.pod
 printf "=head2 The data\n\n" >> report.pod
 
 hdr -r 'A,55:B,5:bf,x3b5:C,18:D,10'  Changes -col -rul -format HTML -display_command_line -start_tag '=begin html' -end_tag '=end html' >> report.pod
 hdr -r 'B,5:bf,x3b5:C,18:D,10'  Changes -col -rul -offset 55 -format HTML -display_command_line -start_tag '=begin html' -end_tag '=end html' >> report.pod
 hdr -r 'B,5:bf,x3b5:C,18:D,10'  Changes -col -rul -offset 55 -offset_start 55 -format HTML -display_command_line -start_tag '=begin html' -end_tag '=end html' >> report.pod

 echo "=head2 Nicely synchronized\n\n" >> report.pod 
 hdr -r 'B,5:bf,x3b5:C,18:D,10'  Changes -col -rul -offset 55 -offset_start 55 -format HTML -display_command_line -start_color bright_yellow -start_tag '=begin html' -end_tag '=end html' >> report.pod
 
 pod2html report.pod > report.html
